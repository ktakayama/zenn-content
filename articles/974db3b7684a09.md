## はじめに

最近、nixの名前を見かける頻度が結構、いややたらと増えてきたなと感じており、何やら流行ってる雰囲気をひしひしと受けています。ただ、自分はパッケージ管理にはHomebrewを使っていて、新しいマシンの初期セットアップ用のスクリプトも用意してこまめにメンテナンスしているので、いわゆるパッケージ管理ツールが新しく増えてもなーと、気になるもののスルーしてました。

そんな怠惰な日々を過ごしてる中、TECH WORLDさんのこの動画に出会うわけです。

https://youtu.be/urpzO1r1Y1c?si=7jKhTY-rpjOyVQ1P

いやはや、これを観たらですね、何かしら刺さるポイントは皆さんあるんじゃないでしょうかね。nix気になる方は必見ですよ！

ちなみに自分は「**すぐ元の環境に戻せるんでとりあえず試して、合わなかったら消せばいい**」という話に、まあ確かになーって思いました。食わず嫌いもよくないよなと。

もうひとつは、**macの設定を宣言的に管理できる**というところ。自分はmacの初期セットアップ用にドキュメントを含めた手順書を用意してるんですけど、システム設定は特に真心込めた手作業なんで、複数マシンで設定を同期管理できてないんですよね。これができるようになったら結構嬉しいなと思いまして。

というわけで、もうこりゃやるしかないなと、ちょっと触ってみることにしました。

## Nixのセットアップ

セットアップは、こちらを参考にさせていただきました。

https://zenn.dev/trifolium/books/1c0373f3570334

記事を読みながらAIと一緒にペア作業でセットアップを進めると、わからないところを確認しながら進められるので結構捗ります。最近こういう作業はすっかりこのスタイルになりましたねー。

インストールスクリプトはいくつかあるようですけど、私は記事にある通りに標準的な方式でやりました。

```sh
curl -sSfL https://artifacts.nixos.org/nix-installer | sh -s -- install
```

セットアップ中に地味に気づかなかったのが、シェルの補完。nixでインストールしたツールの補完が効かないなぁ…と思っていたのですが、どっかで何かの手順が漏れてたかもしれません。自分は`.zshrc` に以下を追加してます。

```sh
export NIX_PREFIX=$HOME/.nix-profile
if [ -d ${NIX_PREFIX}/share/zsh/site-functions ]
then
   fpath=(/nix/var/nix/profiles/default/share/zsh/site-functions/ $fpath)
   fpath=(${NIX_PREFIX}/share/zsh/site-functions $fpath)
fi
```

## `nix run` がめちゃくちゃ便利では？！

ちょっと触ってみて、まず「これは便利では？！」となったのが `nix run` でした。何ができるのかというと、インストールなどの操作をせずにワンライナーでその場で好きなツールを実行できるんですね。`npx` とか `docker run` とかと同じ感じです。`npx` みんな好きですよね。すぐ実行できますもんね。私も好きです。

で、以下の記事に書いたようにまさに自分は `ffmpeg` を `docker run` で使ってるんで、これそのまま `nix run` に乗り換えできる案件だなと。

https://zenn.dev/arm_techblog/articles/13f6d1ba7315bf

インストールするほどでもないよなーっていうソフトウェアが手軽に動かせるのは、Homebrewにはなかった強みですね。ちなみに `ffmpeg` だとこうなる。

```sh
nix run 'nixpkgs#ffmpeg' -- \
    -i input.mov \
    -filter_complex "[0:v]fps=60,scale=320:-1:flags=lanczos,palettegen[p]; \
                     [0:v]fps=60,scale=320:-1:flags=lanczos[v]; \
                     [v][p]paletteuse" \
    -loop 0 \
    -y output.gif
```

## `home-manager` も導入

この勢いでパッケージ管理ツールとしての `home-manager` も試してみました。これも結構いいです。今はHomebrewでインストールしたものを少しずつ移していってます。

特にいいなと思ったのが、**前の状態にすぐ戻せる**という点。

Homebrewのパッケージのアップデート、ほぼ毎日のようにやってるんですけど、本当にごく稀にですが動作がおかしくなる状態になることがあるんですよね。そういうときに、前の状態に戻すのがまあまあ大変というか面倒というか結構時間かかる。これがまた、忙しい合間になんとなくでアプデしたら死ぬみたいな事が、まあ起きる。

それがnixだと、過去の状態にサクッと戻せる。これはかなり心理的に安心感がありますね。

```sh
home-manager switch --rollback
```

## `nix-darwin` …はまだ

ちょっと時間がなくてmacOSの設定が管理できるようになるという `nix-darwin` にはまだ手を出せていないです。若干セットアップや概念、設定が複雑そうな印象があってもうちょっと今の新しい世界観に慣れてからにしようかなと。

## まとめ

ということで、しばらく触ってみての所感としては、もうnixでいいんじゃねって感じに…。nixは原理的に依存関係が衝突しないらしく、依存関係地獄とは無縁みたいなんですよね。最高かよ。そして何より、rollback一発で戻れる安心感。これはもう、知ってしまったら戻れないんじゃなかろうか。

最初は「Homebrewで困ってないし…」と思ってたんですが、触ってみたら別に置き換える話じゃなくて、共存しつつ良いとこ取りができる感じでした。動画で言ってたとおり、合わなかったら消せばいいので、気になってる方はとりあえず試してみるのがおすすめかと思います！

おしまい。
