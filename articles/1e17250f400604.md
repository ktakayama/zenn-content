## はじめに

vimが好きで、基本はターミナルからコーディング作業とかしてるんで、tmuxはもうずっと必須ツールとしてウインドウやセッションはいっぱい作ってバリバリ使ってたんですけど、tmuxやターミナルの画面分割はあんまり必要性を感じてなかったんですよね。というか操作が難しく感じていてあまり使ってませんでした。せいぜいvim内で分割するくらい。

ただ、最近になってAIのコーディングエージェントの利用が増えてきていて、これがですね、画面を分割して並列で走らせる…というやり方がなんだかんだ増えてきているわけです。並列で1つのタスクを実行させるとなると、1タスク1ウインドウで完結させるのが、まあわかりやすいのかなと。

で、重い腰を上げてペイン操作をしっかりできるようになりたいなと思って、まとまった情報がないかなと調べてたんですが、意外とペイン操作に特化した情報が見つからなくてですね。当たり前だけど、入門みたいな記事だとセッションやウインドウの説明がメインだしで、その辺は全然困ってないんだよなあという。

manを見ても、全部のキーバインド設定の中から毎回paneで検索してるので非常に効率が悪く、なんていうか全体感がよくわからない。

というわけで、この記事はtmuxは普通に使ってるけどペイン分割はあまり馴染みがないという人向けに、ペインにフォーカスしてまとめたいなと思った次第で書くことにしました。

## ペインの基本操作

以降の説明でのprefixキーはデフォルトの`C-b`（Ctrl+b）で記載しています。変えてる人は適宜読み替えてください。

### 分割

まずは画面を分割するところからです。デフォルトのキーバインドは以下の2つ。

- `prefix + "` → 上下に分割（水平線で区切る）
- `prefix + %` → 左右に分割（垂直線で区切る）

### 移動

分割したペイン間を移動する方法です。デフォルトで使えるキーバインドがいくつかあります。

- `prefix + Up/Down/Left/Right` → 矢印キーの方向のペインに移動
- `prefix + o` → 次のペインに移動（ペイン番号順に巡回）
- `prefix + ;` → 直前にいたペインに戻る
- `prefix + q` → ペイン番号を表示して、番号を押すとそのペインにジャンプ

`prefix + q`は地味に便利で、ペインがたくさんある時に番号指定で一発で飛べます。

### リサイズ

ペインのサイズを調整したい場面は結構あります。デフォルトのキーバインドは以下です。

- `prefix + C-Up/Down/Left/Right` → 1セル単位でリサイズ
- `prefix + M-Up/Down/Left/Right` → 5セル単位でリサイズ

`C-`はCtrl、`M-`はMeta（Alt/Option）キーです。微調整ならCtrl+矢印、ざっくり変えたいならMeta+矢印、という使い分けですね。`prefix + M-Up`などでリサイズした後、すぐに`M-Up/Down/Left/Right`キーを押すと連続で実行できるので、大きく変えたい時も便利です。

あと、`set -g mouse on`でマウスモードを有効にすると、ペインの境界線をドラッグしてリサイズできるようになるのでこれはかなり便利です。ただ、マウスモードを有効にするとターミナルをクリックしたりスクロールしたりしたときの動作が変わってしまうので好みに応じてという感じですね。自分は使っていません。

## ペインの便利機能

基本操作に加えて、知っておくと便利な機能がいくつかあります。

### ズーム（一時的な最大化）

`prefix + z`で現在のペインを一時的にウインドウ全体に拡大できます。もう一度`prefix + z`を押すと元のレイアウトに戻ります。

「ちょっとこのペインだけ広く見たいな」って時に便利です。分割したまま一時的に全画面にできて、すぐ戻せるのは良いですね。

### ペインとウインドウの相互変換

ペインで作業してたけど「やっぱりこれ独立したウインドウにしたいな」ってことがあります。逆もあります。

- `prefix + !` → 現在のペインを新しいウインドウとして切り離す
- `join-pane` → 別のウインドウをペインとして取り込む（デフォルトのキーバインドなし）

`join-pane`にはデフォルトのキーバインドが割り当てられていないので、`prefix + :`でコマンドモードに入って実行します。たとえば現在のセッションのウインドウ2を現在のウインドウにペインとして持ってくるなら以下です。

```
join-pane -s :2
```

他のセッションのウインドウも指定できるんですけど、パラメータがだいぶ複雑になるので毎回手入力するのは面倒です。次に紹介するマーク機能を使うとかなり楽になります。

### ペインのマーク

`prefix + m`で現在のペインに「マーク」をつけられます。`prefix + M`でマークを解除します。マークは同時に1つのペインにしかつけられません。

マークしたペインは`join-pane`や`swap-pane`で`-s`を省略した時に自動的に対象になります。つまり移動したいペインで`prefix + m`しておいて、移動先で`join-pane`するだけで持ってこれるのです。別のセッションのペインにもマークできるので、セッションをまたいだ操作も簡単です。

### ペインの入れ替え・回転

ペインの配置を入れ替えたい時に使います。

- `prefix + {` → 現在のペインを前のペインの位置と入れ替え
- `prefix + }` → 現在のペインを次のペインの位置と入れ替え
- `prefix + C-o` → すべてのペインを順方向に回転
- `prefix + M-o` → すべてのペインを逆方向に回転

### レイアウトの切り替え

ペインのレイアウトを一発で変更できます。`prefix + Space`を押すと、用意されたレイアウトを順番に切り替えていけます。`prefix + M-数字`で特定のレイアウトを直接指定できます。

## 外観のカスタマイズ

ペインを使い始めると、デフォルトの見た目だと境界線がわかりにくかったり、どのペインがアクティブなのかパッと見でわからなかったりするんですよね。この辺は設定でけっこういじれます。

### ボーダーのスタイル

ペインの境界線の色は`pane-border-style`と`pane-active-border-style`で設定します。

```
# 非アクティブなペインのボーダー
set -g pane-border-style 'fg=colour238'

# アクティブなペインのボーダー
set -g pane-active-border-style 'fg=colour39,bold'
```

デフォルトだとアクティブなペインのボーダーが緑っぽい色になっていたりするんですけど、ターミナルの配色テーマによっては見にくいことがあります。自分の環境に合わせて調整するのがいいですね。

### ボーダーの線種

`pane-border-lines`でボーダーの線の種類を変更できます。

```
set -g pane-border-lines single    # デフォルト
set -g pane-border-lines double    # 二重線
set -g pane-border-lines heavy     # 太線
set -g pane-border-lines simple    # ASCII文字（+, -, |）
set -g pane-border-lines number    # ペイン番号を表示
```

`number`はボーダー自体にペイン番号が表示されるのでちょっと面白いです。文字サイズなど環境によって見え方も変わってくるので、色々試してみるのが良いかなと思います。

### ペインのボーダーにステータスを表示

`pane-border-status`を有効にすると、ペインのボーダー上にテキスト情報を表示できます。

```
# ボーダーの上側に表示
set -g pane-border-status top

# ボーダーの下側に表示
set -g pane-border-status bottom

# 無効化（デフォルト）
set -g pane-border-status off
```

表示内容は`pane-border-format`で指定します。

```
set -g pane-border-format ' #P: #{pane_title} '
```

`#P`はペイン番号、`#{pane_title}`はペインのタイトルです。他にも`#{pane_current_command}`で実行中のコマンド名を表示したりできます。ペインが増えてきた時にどのペインで何をやっているかが一目でわかるので、地味に助かる機能ですね。

### アクティブペインの識別方法

`pane-border-indicators`でアクティブペインの識別方法を変えられます。

```
set -g pane-border-indicators colour   # 色で区別（デフォルト）
set -g pane-border-indicators arrows   # 矢印で表示
set -g pane-border-indicators both     # 色と矢印の両方
```

## キーバインドのカスタマイズ

ここではよく見かけるキーバインドのカスタマイズパターンを紹介します。

### 分割キーを変更する

分割用のキーは見た目で方向がわかる`-`と`|`に変更するのが割と定番そうです。ついでに`-c "#{pane_current_path}"`をつけておくと、新しいペインが現在のディレクトリで開くようになるのでおすすめです。

```
bind - split-window -v -c "#{pane_current_path}"    # 上下に分割
bind | split-window -h -c "#{pane_current_path}"    # 左右に分割
```

### Vim風にペイン移動

vim使いならこれは入れたいですよね。`hjkl`でペイン間を移動できるようにします。

```
bind h select-pane -L
bind j select-pane -D
bind k select-pane -U
bind l select-pane -R
```

### Vim風にリサイズ

リサイズもvimキーで。`-r`をつけるとprefixキーなしでリピート入力できるので、連続でサイズ調整する時に楽です。

```
bind -r H resize-pane -L 5
bind -r J resize-pane -D 5
bind -r K resize-pane -U 5
bind -r L resize-pane -R 5
```

### prefixなしでペイン番号指定ジャンプ

`bind`に`-n`をつけるとprefixキーなしで直接キーが効くようになります。`Meta+数字`でペイン番号を指定して直接ジャンプする設定で、`prefix + q`で移動するよりも速いです。

```
bind -n M-0 select-pane -t 0
bind -n M-1 select-pane -t 1
bind -n M-2 select-pane -t 2
bind -n M-3 select-pane -t 3
bind -n M-4 select-pane -t 4
```

ペイン番号が0オリジンだから、0だけ離れてるのがいまいちなんですよねー。

## まとめ

というわけで、tmuxのペイン分割、使ってみると思ったより難しくなかったです。セッションやウインドウを使いこなしている人なら覚えることはそこまで多くないですし、今回自分用に使えそうなキーバインド一覧も作ったので、あとは使いながら慣れていくだけかなと。

外観まわりの設定も充実しているので、ペインの境界が見にくいとかアクティブがわかりにくいとか、そういうことがあれば設定で解消できます。

特にAIエージェントを並列で走らせるようなワークフローだと、ペイン分割はほんとに便利なので、同じような状況の人はぜひ試してみてほしいです。
